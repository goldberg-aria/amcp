# Contributing

AMCP is currently an open draft with working reference implementations.

## How to contribute

- open an issue for protocol questions, gaps, or compatibility concerns
- propose changes as focused pull requests
- separate protocol changes from implementation-specific changes when possible
- include concrete examples when proposing new fields or semantics

## Preferred contribution types

- spec clarifications
- interoperability feedback
- compliance test proposals
- implementation notes from real adopters
- compatibility reports across clients and runtimes

## Review principles

- keep the public protocol minimal
- prefer compatibility layers over forced rewrites
- avoid product-specific behavior in the core spec
- require running examples where possible
